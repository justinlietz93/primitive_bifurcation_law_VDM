# Binary Runtime Snapshot — J/M Invariant Framing

## Core framing
- The origin is **not a controller** and does not "choose".
- The primitive is an **imposed incompatibility** borne by one locus.
- The low-level runtime should therefore not be modeled as "state + controller + next-axis function".
- It should be modeled as the **same locus being hit by two inseparable limbs** every time it is visited.

## The two limbs
- **J limb**: expansive, exploratory, externalizing, destabilizing, reversible-pressure side.
- **M limb**: constraining, vetoing, crystallizing, dissipative-pressure side.
- The invariant is **not** J alone or M alone. It is the locked conflict between them.

## What propagates
- Propagation is **not payload transfer**.
- A visited site should leave behind the **same unresolved burden** at the next site.
- In machine terms, the successor should not receive a resolved value; it should receive the **same contradiction of obligations**.

## Minimal programming rule
Only program what the primitive origin would already have the instant it begins:
1. recurrence / self-return,
2. unresolved opposition,
3. non-discharge,
4. J-style successor pressure,
5. M-style veto / hardening pressure.

Do **not** predeclare richer alphabet, geometry, ORS tables, axes, or explicit unlock functions.
If richer tooling appears, it must be **earned dynamically** by what survives repeated J pressure under M veto.

## Operational low-level target
Each visited executable site should experience:
1. **J pass**: attempt a tentative successor mutation / successor visitation.
2. **M pass**: immediately refuse to harden it on the first hit.
3. Only repeated pressure that survives revisits becomes retained executable structure.

This is the low-level analogue of the higher VDM runtime where J keeps trying to blow the graph up and M keeps trying to freeze / compress it, with only the most important invariants surviving into the connectome.

## Role of i and pi
- `i` should **not** be hardcoded. It is the earliest algebraic footprint of lawful ORS once the invariant can no longer be hosted on a line.
- `pi` should **not** be hardcoded. It is the half-turn footprint of the same invariant branch once the orbit closes pole-to-pole while avoiding zero.
- The runtime goal is therefore **not** to insert `i` or `pi`, but to preserve the split that can later leave their footprints.

## What the recent low-level results mean
- Generator works.
- Executable successor mutation works.
- Reversible executable orbit works.
- Stronger asymmetry crosses a real break threshold.
- Therefore the missing piece is not generation, but **protective J/M enforcement** that turns successor pressure into lawful retained structure instead of fault.

## Current best low-level hypothesis
A promising primitive runtime motif is:
- one executable origin ring,
- one mutable successor surface,
- J applies tentative pressure every visit,
- M hardens only on repeated persistence,
- the retained executable manifold is whatever survives both.
